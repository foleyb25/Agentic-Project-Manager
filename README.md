# AgenticProjectManager (APM)

A configurable, project-agnostic **agentic project-manager pipeline**, packaged
as a **Claude Code plugin** — no application code, just skills, agents, hooks,
and MCP config. One PM (the main Claude Code process) orchestrates; **advisory
role agents** — generated to fit *your* project — inform its decisions. Humans
stay the execution layer.

- Building a game? Tailoring produces a Game Designer, Gameplay Engineer,
  AI Engineer, Tech Artist, QA.
- Building an ERP system? Tailoring produces Finance, HR, QA, Compliance, etc.

The PM reads your spec, consults the roles, writes deduplicated GitHub Issues,
logs every action to a Supabase ledger, and replies on every thread that
mentions `@pm`. It runs nightly, on `@pm` comment events, and on manual
dispatch.

## How it works — native Claude Code primitives only

| Piece | Mechanism |
|---|---|
| PM procedure (the deterministic loop) | `skills/pm-run/SKILL.md`, invoked as `/pm-run <trigger> <run_id>` — read spec → read ledger state (where you left off) → sweep comments → analyze sentiment → consult → write deduped tickets → log receipts → reply → advance cursor |
| Role consultants | `agents/*.md` subagents; frontmatter `tools:` is the permission boundary (read-only + explicit `mcp__server__tool` grants), `model:`/`maxTurns:` the cost levers |
| Project facts | `PROJECT.md` (generated by `/tailor` from your `SPEC.md`): roster, routing table, labels, milestone, spec paths |
| GitHub + ledger | MCP servers in `.mcp.json` — hosted GitHub MCP, Supabase MCP (`execute_sql` with idempotent upserts) |
| Guardrails | `hooks/hooks.json` — e.g. a PreToolUse guard that blocks any destructive SQL before it reaches the ledger |
| Packaging | `.claude-plugin/plugin.json` — load with `claude --plugin-dir AgenticProjectManager` (works as a git submodule) |

Design rule: **procedure in skills, boundaries in frontmatter + hooks,
judgment in the model.**

## Quickstart (new project)

1. **Fork this template for your project.** Each project gets its own fork —
   it holds the tailored PROJECT.md/agents, and it's what CI pulls. Keep the
   template as an upstream remote to pull improvements:
   ```bash
   git remote add template git@github.com:foleyb25/Agentic-Project-Manager.git
   git fetch template && git merge template/main
   ```
2. **Add your fork to your project** as a submodule:
   ```bash
   git submodule add git@github.com:<you>/Agentic-Project-Manager-<project>.git AgenticProjectManager
   ```
3. **Write `SPEC.md`** at your project root: vision, roadmap/milestones, team
   roles, process rules. The richer the spec, the smarter the tailoring.
4. **Tailor** — from the project root:
   ```bash
   claude --plugin-dir AgenticProjectManager
   > /tailor
   ```
   Review the generated `PROJECT.md` + `agents/*.md`, commit + push to your
   fork, bump the submodule pin.
5. **Ledger**: create a Supabase project and apply
   `supabase/migrations/*.sql` (SQL editor). Note the project ref.
6. **Bootstrap GitHub** (labels + milestone): `/bootstrap` in the same session.
7. **Wire up CI**: copy `workflows/pm.yml` → your project repo's
   `.github/workflows/pm.yml`; set Actions secrets: `ANTHROPIC_API_KEY`,
   `GITHUB_MCP_TOKEN` (PAT — the Actions token can't auth the hosted GitHub
   MCP), `SUPABASE_ACCESS_TOKEN`, `SUPABASE_PROJECT_REF`.
8. **Smoke test**: Actions → pm-agent → Run workflow; then comment `@pm hello`
   on an issue and watch the reply + 👀 land.

## Running locally

```bash
cp AgenticProjectManager/.env.example AgenticProjectManager/.env  # fill in yourself
AgenticProjectManager/bin/pm-local.sh            # one PM run (subscription login)
# Windows: .\AgenticProjectManager\bin\pm-local.ps1
```

Local runs use your interactive `claude` login — a subscription seat bills no
API tokens. CI uses `ANTHROPIC_API_KEY`.

## Cost knobs

- Per-role `model:` frontmatter — consultations on a cheaper model.
- `maxTurns:` per agent; `--max-turns` / `--max-budget-usd` per run
  (workflow `claude_args`).
- The pm-run skill front-loads ledger/cursor reads so the PM doesn't burn
  turns rediscovering state.

```
docs/setup.md                one-time setup checklist
workflows/pm.yml             GitHub Actions workflow template
PROJECT.md                   tailored project facts (created by /tailor)
```
