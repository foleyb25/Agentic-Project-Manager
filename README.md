# AgenticProjectManager (APM)

A configurable, project-agnostic **agentic project-manager pipeline**. One PM
agent orchestrates everything; a set of **advisory role agents** — generated to
fit *your* project — inform its decisions. Humans stay the execution layer.

- Building a game? Tailoring produces a Game Designer, Gameplay Engineer,
  AI Engineer, Tech Artist, QA.
- Building an ERP system? Tailoring produces Finance, HR, QA, Compliance, etc.

The PM reads your spec, consults the roles, writes deduplicated GitHub Issues,
logs every action to a ledger, and replies on every thread that mentions
`@pm`. It runs nightly, on `@pm` comment events, and on manual dispatch.

## How it fits into a project

```
your-project/
├── SPEC.md                  ← your project spec (replace the placeholder!)
├── <your source dirs>/      ← e.g. UnrealProject/, backend/, app/
├── AgenticProjectManager/   ← this repo, as a git submodule
└── .github/workflows/pm.yml ← copied from workflows/pm.yml here
```

The PM agent runs with the **project root as its working directory** — it reads
`SPEC.md`, your code, and your issue tracker; it writes only GitHub
issues/comments and ledger rows. Role agents are read-only (any exception, e.g.
QA filing bugs, is an explicit scoped grant in config).

## Quickstart (new project)

1. **Add APM to your project** (as a submodule, so the version that runs in CI
   is pinned by your project repo):
   ```bash
   git submodule add https://github.com/<you>/AgenticProjectManager.git AgenticProjectManager
   ```
2. **Write `SPEC.md`** at your project root: vision, roadmap/milestones, team
   roles you want, process rules. The richer the spec, the smarter the
   tailoring.
3. **Tailor** — open Claude Code anywhere in the project and run:
   ```
   /tailor
   ```
   Claude reads `SPEC.md` and generates `config/apm.yaml` (roles, routing
   table, label taxonomy, milestones) and one `agents/roles/<role>.md` charter
   per role. Review, adjust, commit (to your APM fork — the tailored source is
   what CI pulls).
4. **Bootstrap GitHub** (labels + first milestone):
   ```bash
   uv run apm bootstrap
   ```
5. **Wire up CI**: copy `workflows/pm.yml` to your project repo's
   `.github/workflows/pm.yml`; add repo Actions secrets:
   `ANTHROPIC_API_KEY`, and for the Supabase ledger `SUPABASE_URL` +
   `SUPABASE_SERVICE_KEY` (apply `supabase/migrations/*.sql` to your Supabase
   project first). No Supabase? Set `ledger.backend: jsonfile` — zero external
   services (see Ledger backends).
6. **Smoke test**: Actions → pm-agent → Run workflow. Then comment `@pm hello`
   on any issue and watch it reply.

## Running locally

```bash
cd AgenticProjectManager
uv sync
uv run apm run --trigger manual            # one full PM run (uses your local claude login)
uv run apm consult ge "pipeline.ci: sanity-check the workflow"
uv run apm ledger get sync_cursor scope=repo
uv run apm bootstrap
```

Local runs use your interactive `claude` login — if that's a Claude
subscription seat, local PM runs don't bill API tokens. CI runs use
`ANTHROPIC_API_KEY`.

## Cost knobs

- `runner.engine: cli` (default) wraps `claude -p`; `sdk` uses the Python
  Claude Agent SDK (`uv sync --extra sdk`) — same prompts either way.
- `runner.consult_model` / per-role `model:` in `config/apm.yaml` — run
  consultations on a cheaper model than the PM.
- `runner.max_turns` and `runner.max_budget_usd` — hard per-run ceilings.
- Prefetch: `apm run` pre-fetches comments + ledger state into
  `.apm/run_context.json` so the PM doesn't burn turns on discovery.

## Ledger backends

The ledger (see `docs/PM_DATASTORE.md`) makes the PM idempotent: never analyze
a comment twice, never open a duplicate issue, keep an audit trail per run.

- **`supabase`** (default): Postgres via PostgREST. Apply
  `supabase/migrations/*.sql`; the service-role key is held only by the PM
  (CI secret). Humans/other agents read via Supabase Studio / anon key.
- **`jsonfile`**: a local JSON file (`.apm/ledger.json` by default) — zero
  services. In CI, persist it across runs with `actions/cache` (see the
  commented block in `workflows/pm.yml`).

## Repo map

```
agents/pm.md           PM system prompt (generic — do not project-tailor)
agents/roles/*.md      role charters (generated by /tailor)
config/apm.yaml        project config (generated by /tailor)
schemas/consultation.json  structured report every role must return
src/apm/               the deterministic protocol: CLI, runners, ledger, dedup
workflows/pm.yml       GitHub Actions workflow template for project repos
supabase/migrations/   ledger DDL + RLS
docs/PM_DATASTORE.md   ledger schema + processing protocol
docs/setup.md          one-time setup checklist
```

**Design rule: protocol in Python, judgment in prompts.** Dedup, hashing,
cursor management, ledger I/O, and guardrails are code; comment analysis,
consultation synthesis, and ticket writing are Claude.
